2. QR Codes

The QR Label you see above is a key component of the proposed ccFam System.

The image is created from an Avery sticker sheet:
Square Avery Lbls 1x1 (PrestaÂ® 94103; 48 labels per sheet)

These QR codes can be used for all entities in the system, including:

  • Children
  • CareGivers
  • Facility Administrators
Using these QR labels:
  • Makes work easier for Facility Admins (they only have to scan the sticker and click on "Child CheckIn" to register a child)
  • Reduce the possiblity of error
  • Have unrelated advantages (e.g.: If children place these on all their belongings; if they forget their phone or water-bottle at the facility the Admin will know instantly by scanning the QR Code.)
  • If used by the parents while picking up, there is an additional level of security added.
  • In case of a medical emergency for a child, any adult can instantly pull up the child's Emergency Information including Doctor's contact info and allergy information.

2. Project Workflows

This section of the proposal outlines the workflows for tasks related to each of the principal ccFam users, viz.: Facility Administrators and the System Administrator.

The entire workflows are listed for clarity and to reduce the possiblity of error, and to ensure that the vendor and organization are in sync about every Functional Aspect of the new system.

Child Workflows

ccFam website

The main ccFam website will have three sections for visitors:

  1. Child/Parent Portal
  2. CareGiver Portal
  3. Site Admin Portal
  • Each of these sections will allow the usual website flows for registration/login/lost password recovery, etc.
  • We will offer to let the user use Social Media and Email platforms (e.g. Outlook/Google/FaceBook/X) to login. This is standard procedure nowadays to allow users a one-click login without entering a username/password.
  • The website will offer Password recovery and 2FA (Two Factor Authentication, which allows the use of Voice Mail and/or SMS to verify identity)
Besides this, the ccFam website will contain the usual site-oriented information (find a facility, lookup a facility map/address, contact us, etc.)

CheckIn

  1. Child arrives at facility
  2. The Facility Adminstrator scans the child's QR code (from a sticker)
  3. The system pulls up the child's record on the FacAdmin's device (showing name and photo)
  4. After confirming, FacAdmin clicks on the CheckIn button
  5. The System prompts the FacAdmin to assign the child to a Room (only rooms for the current facility are shown)
  6. The child is checked-in, and the SysAdmin's dashboard count increases by one.
Roster
  1. At some time during the morning - after the bulk of the children have arrived - the FacAdmin clicks on a button to print out all Rosters for the facility
  2. The Roster contains the following data:
    • Facility Name
    • Room name/id
    • One row for each child in the Room, with columns for |Photo|Name|Age|QR Code
    • To reAssign a child to another room the FacAdmin just scans the QR Code and clicks on the 'reAssign' button. If a child is assigned to a different room the system updates the counts for both rooms live.
    • The system automatically caculates the ratios required by DCF and updates the dashboard in real-time

CheckOut

  1. Similar to above, except that since the system knows that the child is currently at the facility, the CheckIn button is auto-disabled.

Additionally we will offer live updates on Child-CheckOuts; this may be useful in situations where one parent tells the other "Pick up the child if I don't make it to the childcare center".

SysAdmin Workflows

The ccFam System will offer the System Administrator all capabilities to view/edit/delete any of the tables:

  • Child
  • Facility
  • CareGiver
  • FacAdmin

The system will of course also allow the SysAdmin to communicate effectively with all users of the system.

We shall create eMail Templates for all common usage patterns, e.g.: Facility Adminstrators will have the option of filling a "Lost Item email" which allows them to just (only) enter the description of the item ("A Cinderella water bottle") and with just one click, send the email message to the parent.

However, the main module of interest to SysAdmins is the Dashboard.

The System Dashboard will update multiple times a second and show live streaming data from each facility including totals and ratios.

Please note that clicking on any of the bottom sections (e.g. "In Violation") will show you detailed information.

Also, clicking on the table shown in the "In Violation" section will bring up the details of the specific facility you click on (right now it only shows a demo screen but the system shall show data for each row)

As you can see, the details include a FloorPlan of the facility (with the specific location of the room highlighted)

In addition to this functionality, when the SysAdmin clicks on the "Invite Caregiver" button the system:

  • Auto-filters the Caregivers and shows only the ones with the experience needed (age-groups etc.)
  • Allows the SysAdmin to send a one-click Invite
  • The Caregiver will recieve a VoiceMail or SMS asking to accept/decline and will be given a timeout (say 15 minutes, depending on your feedback)
    This is in order to allow the CareGiver to lookup the facility or decide whether the engagement is convenient.
  • The system shall display live information status: if the CareGiver doesn't reply it moves to Yellow (awaiting reply) else accept/reject allows the SysAdmin to take the next step.
  • The facility remain in yellow status until the additional caregiver checksIn.
  • The SysAdmin shall be able to look at all invites sent and status for each (accepted/rejected/pending).
We believe that a system designed in such a manner will be very intuitive to use and, therefore, the most productive.

Of course, we shall be totally open to any modifications you prefer to make to these workflows and are confident that we shall be able to translate all your business requirements into a top-notch software product.

4. Technical Approach

We intend to use the following Tech Stack to build the ccFam System:
BackEnd: Microsoft dotNet + FSharp
Database System: MongoDb
Web Server Hosting: Microsoft Azure
Authentication System: Microsoft Azure/AD
VoiceMail/SMS: Twilio

These are all industry-standard vendors offering the most reliable and business-proven technology available today.

A Note about Scale
ccFam shall be built from scratch with a view to scale. By that we mean that the system shall be able to easily handle a great many facilities (lets say a hundred would be no problem).

Expanding the product to meet really large scale demands (say, 100m children: the entire US population) would also not require any basic architecture changes, but we'll then need to optimize for performance: handling tens of millions of checkIns simulataneously would slow down the system; so to prevent that we would need to set up support systems and backup devices.
